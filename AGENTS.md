# AGENTS.md — citeproc-plus

`citeproc-plus` bundles `citeproc-js` with Citation Style Language (CSL) styles
and locales into a single npm package used by Fidus Writer's bibliography and
citation rendering.

## Repository layout

| Path | Purpose |
|------|---------|
| `src/` | TypeScript source (`index.ts`, `tools.ts`). |
| `build/` | Downloaded CSL styles/locales and converted assets. Generated by the build. |
| `dist/` | Published JavaScript bundles and source maps. Generated by the build. |
| `convert.py` | Python script that converts downloaded CSL styles/locales into the bundled assets. |
| `rollup.config.ts` | Rollup configuration for ESM and CJS builds. |
| `test/` | Vitest tests. |

## Build

The full build downloads the latest styles/locales from GitHub, converts them,
and produces the bundles:

```bash
npm run build
```

To rebuild only the TypeScript bundles without re-downloading styles/locales:

```bash
npm run rollup
npm run build:types
```

## Test

```bash
npm test
```

## Lint

```bash
npm run lint
```

## Release checklist

1. Make the source changes.
2. Run `npm run build` and `npm test`.
3. Bump the version: `npm version patch|minor|major`.
4. Publish: `npm publish`.
5. Push commits and tags: `git push && git push --tags`.
6. Update `citeproc-plus` in consuming repositories, such as
   `fiduswriter/fiduswriter/document/package.json5`, and run
   `./manage.py transpile --force` in the main Fidus Writer app.

## Source maps

Rollup emits source maps. The `sourcemapFileNames` option prefixes every map
with `citeproc-plus-` and includes a content hash so that bundled source-map
URLs are unique and do not clash with maps from other packages in Fidus Writer.
